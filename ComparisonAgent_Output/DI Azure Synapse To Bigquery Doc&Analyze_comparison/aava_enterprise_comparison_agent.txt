**APPENDIX A: TECHNICAL REFERENCE**

Comparison Methodology Summary:
- Files were compared using SHA-256 hashing and line-by-line diff analysis.
- Match percentages were calculated from unchanged line proportions.
- Dimension scores were derived from systematic evaluation across ten quality aspects.
- All calculations used deterministic formulas for reproducibility.

Dimension Scoring Reference:
1. Business / Functional Logic: Core business rules, calculations, outcomes
2. Code / Syntax Accuracy: Technical correctness, standards
3. Data Flow & Transformations: Data processing steps, joins, filters
4. Conditional / Control Logic: IF/ELSE, loops, branching
5. Error Handling Strategy: Exception handling, validation
6. Performance Patterns: Efficiency, optimization, scalability
7. Unit Test Coverage: Test completeness, coverage breadth
8. Output Structure & Modularity: Organization, maintainability
9. Documentation Quality: Comments, docstrings, explanations
10. End-to-End Functional Equivalence: Overall functional alignment, outputs

Match Percentage Formula:
```
Match % = (Unchanged Lines / Total Lines) × 100
```

Decision Threshold Reference:
- PASS: >= 90%
- CONDITIONAL PASS: 75-89%
- FAIL: < 75%

File Matching Methods Used:
- Level 1: Direct name match
- Level 2: Role/keyword recognition
- Level 3: Content similarity (>=70%)
- Level 4: Alphabetical pairing

This appendix provides a technical foundation for the comparison methodology, supporting audit and reproducibility.

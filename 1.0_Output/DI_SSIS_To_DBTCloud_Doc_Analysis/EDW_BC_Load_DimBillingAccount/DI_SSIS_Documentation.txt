1. Overview of Package

Purpose:
The SSIS package "EDW_BC_Load_DimBillingAccount.dtsx" is designed to extract, transform, and load (ETL) data from Guidewire source systems into the DimBillingAccount dimension table in the enterprise data warehouse (EDW). It performs incremental data loads, handles inserts and updates, and ensures the dimensional data is current and accurate for downstream analytics and reporting.

Broader Data Integration Strategy:
This ETL process is a critical component of the EDW, supporting subject area data marts and enterprise reporting. By maintaining the DimBillingAccount dimension, it enables accurate customer, billing, and account analytics, supporting regulatory, operational, and business intelligence needs.

Business Use Case:
The package addresses the need to track and analyze billing account information, including account hierarchies, types, segments, service tiers, and address details. It supports business processes such as customer segmentation, delinquency management, and service tier analysis.

High-Level Package Components:
- Control Flow: Orchestrates the ETL process, including initialization, main data flow, and process completion/failure handling.
- Data Flow: Extracts data from Guidewire, applies transformations, performs lookups, splits data for inserts/updates, and loads the DimBillingAccount table.
- Variables: Tracks row counts for inserts, updates, unchanged records, and batch processing.
- Event Handlers: Captures errors and logs them for operational monitoring.

2. Package Design and Structure

Design and Modular Layout:
- Sequence: The package starts with process initiation, executes the main data flow, and concludes with success or failure logging.
- Main Data Flow Task: "DFT - Load DimBillingAccount" contains all ETL logic.

Key SSIS Components:
- Source: OLE DB Source ("OLE_SRC - GuideWire") extracts data from Guidewire using a complex SQL query with joins and filters for incremental loads.
- Lookup: "LKP - DimBillingAccount" matches incoming records to existing DimBillingAccount records on PublicID and BeanVersion.
- Derived Column: "DRV - BatchID & Legacy" adds/updates BatchID and LegacySourceSystem columns.
- Conditional Split: "CSPL - Check BeanVersion" determines if a record is new, changed, or unchanged based on BeanVersion.
- Row Count: Multiple row count components track processed, inserted, updated, and unchanged records.
- Destination: "Load DimBillingAccount" (OLE DB Destination) inserts new records; "CMD - Update DimBillingAccount" (OLE DB Command) updates changed records.
- Control Flow Elements: Execute SQL Tasks for process initiation and conclusion, and error event handlers.

3. Data Flow and Transformation Logic

Data Flow Paths and Transformations:
- Extraction: OLE DB Source extracts distinct billing account records with all necessary joins for parent account, insured info, address, and status.
- Derived Column: Adds BatchID and sets LegacySourceSystem to 'WC'.
- Lookup: Matches on PublicID and BeanVersion to identify new/changed/unchanged records.
- Conditional Split:
  - New records (no match in lookup) → Insert path.
  - Changed records (BeanVersion mismatch) → Update path.
  - Unchanged records (BeanVersion match) → Unchanged path.
- Data Cleansing/Validation: Data type conversions (e.g., CAST to VARCHAR), null handling, and business logic for IsActive flag.
- Row Counts: Track the number of source, inserted, updated, and unchanged records for audit and monitoring.
- Data Type Conversions: Explicit CASTs in SQL for all string columns, ensuring consistency with target schema.

5. Key Outputs

- Inserts: New DimBillingAccount records are inserted into the EDW when no matching PublicID and BeanVersion are found.
- Updates: Existing records are updated when the incoming BeanVersion differs, ensuring the dimension reflects the latest source data.
- Unchanged: Records with matching BeanVersion are counted but not modified.
- Row Count Variables: InsertCount, UpdateCount, UnChangeCount, SourceCount are tracked for audit and monitoring.
- Outputs align with business goals by ensuring the DimBillingAccount dimension is accurate, supporting reliable reporting, analytics, and regulatory compliance.
